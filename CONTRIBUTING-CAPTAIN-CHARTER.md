## 设计哲学

> 为爱发电，不设绩效考核。只要是好东西，愿意分享，就值得认可。

本章程的核心目标是**降低参与门槛、分散审核压力、保证内容质量**，让每一位真心想贡献的学弟学妹都能无痛上船。

---

## 三层身份体系

| 层级 | 称号 | 门槛 | 权限 | 审核 |
|------|------|------|------|------|
| L1 | 船员（Contributor） | Fork 仓库并在本地使用 | 提交 Issue、参与 Discussions | 无需审核 |
| L2 | 见习舰长（Captain-in-Training） | 完成**单科**或**局部模块**的整体内容上传 | 提交 PR、进入审核队列 | 1 位正式舰长或 Admiral 审核通过即可 |
| L3 | 正式舰长（Captain） | 累计 3 次审核通过，或独立维护一个子模块 | Reviewer 身份、审核他人 PR、组织决策投票权 | 由 Admiral 授予 |

---

## 晋升路径

### L1 → L2：见习舰长认证

**提交标准**（满足任意一条即可）：

- 完成某一科目（如高数/线代/英语/专业课）的完整笔记整理并打包上传
- 完成某一局部模块的整体内容（如「多元函数积分学」「自动控制原理第五章」）的整理上传
- 提交形式：Fork → 本地整理 → 在提交说明中标注 `submission: 科目/模块名`

**审核标准**：

| 检查项 | 说明 |
|--------|------|
| 格式规范 | 符合母舰的 frontmatter 元数据标准（`generation`, `captain`, `subject`） |
| 隐私脱敏 | 不含个人身份信息、考研报名号、准考证号等 |
| 版权合规 | 不含名师 PDF、盗版讲义、未经授权的扫描件 |
| 内容质量 | 有实际参考价值，非纯复制粘贴（允许 AI 辅助，需人工审校） |

**审核人**：当前由 Admiral（Leo）或任一正式舰长执行。未来由对应科目的正式舰长分流审核。

### L2 → L3：正式舰长认证

**授予条件**（满足任意一条即可）：

| 路径 | 说明 |
|------|------|
| 质量路径 | 累计 3 次 PR 审核通过，证明持续输出高质量内容 |
| 模块路径 | 独立承担一个子模块的日常维护（如英语语料库、高数错题集） |
| 提前认证 | 未上岸但笔记质量极高、社区贡献突出，由 Admiral 直接授予 |

**默认路径**：27 届上岸后，所有见习舰长自动晋升为正式舰长。

**权限**：

- GitHub Reviewer 身份（可 Approve / Request Changes）
- 对应科目的 PR 优先审核权
- 组织重大决策的投票权（如章程修改、新科目开辟）
- 获得 `Captains` Team 成员资格

## FAQ

**Q: 我没有上岸，能当见习舰长吗？**
A: 可以。只要内容质量达标，上岸不是硬性门槛。上岸只是默认晋升正式舰长的触发条件。

**Q: 我只整理了一章的内容，够资格吗？**
A: 够。只要是一个完整的局部模块（如一章、一个专题），而非零散的零散笔记，就符合提交标准。

**Q: 审核周期多久？**
A: 当前 Admiral 单人审核，承诺 7 天内响应。未来由科目舰长分流后，目标 3 天内。

**Q: 成为正式舰长后，还能提交自己的内容吗？**
A: 当然可以。正式舰长的额外权限是审核他人，不代表自己不能继续贡献。
